Some hints on how you can navigate this website

Introduction

This help page will walk you through what the website has to offer, additional explanations can be found on individual pages and clicking info buttons.

We will start from the navigation bar at the top of each view, then look at contents of each of the menus there in more details, discussing each list, item, resource and index view.

All this changes often, because we try to keep this Research Environment as flexible as it needs to be. For a discussion of how this website works, please see the links, presentations and bibliography below.

If you are looking for information about the project, please see our institutional project website.

This help page is all in one page, so you can search through it using your browser search function. The main parts are always accessible form the side bar.

Note that the data as well as the website are in continuous development. You are likely to encounter errors. If you care enough, please, do let us know and we will try to sort them as soon as possible or inform you on why this happens at least.

Navigation Bar

Login

You do not need to login. Only if you are an editor which uses the web based editor, then you might want to login. All content is served to you in its latest version, without any need to login.

Hi guest!

In this menu you find links to some documentation and general project information, especially the team, where you can see each contributor to the resources you are looking at and also navigate directly to a list of their contributions. Want to be among them? You can! The Guidelines have a page to get you started and also some training materials. There you will also get to know when there will be a next workshop.

Clavis

Under this menu you find links to three list views which relate to the Clavis Aethiopica, an ongoing repertory of all known Ethiopic Textual Units. You can use the pprovided CAe numbers to refer univocally to a specific text in your publications. Please note that this CAe reference number shares only the numeric part with the Textual Unit Record Identifier.

You can find more details on each of these lists and their visualization below. The three lists are:

  • Textual Units will return a list of what is most commonly thought of as a literary Work.
  • Narrative Units will return all identified Narrative Units. This are still very few.
  • Documents Corpora is a reproduction of a list view from the EMA project led by Anais Wion which lists encoded documents under archives.

Manuscripts

Under the manuscript menu you will find several ways to access lists of the available manuscripts.

  • Manuscripts (search) will lead you to a search form which will then list the manuscripts from the database with some information (more below)
  • Shelf marks (full list) will return a list of all repositories we know of and for each of them a list of shelfmarks grouped by collection. If you know your shelfmark and you like scrolling, this is where to look. If you use the search without setting any filter, this will also look in shelf marks and get you to possible options.
  • Manuscript Images will load a Mirador viewer open on the list view, with all images of manuscripts we are currently serving or we have a usable link to a manifest for. You can either directly view the images in this mirador instance or click on the info box and go to the record for that manuscript.
  • Catalogues will provide you with a list of all publications which are listed in the Catalogue Bibliographies, and clicking on one of them will open a list view with all the manuscripts in that catalogue for which we have a record.

Places

Under the Places menu you can get to two different list views.

  • Places will lead you to a list view of all place-like records (including repositories).
  • Repositories will lead you to a list view of the repositories only and tell you how many manuscripts are kept there.

Indexes

The links listed under this menu will lead to additional views of the data according to specific needs. Each of these is explained in more detailes below in a specific section.

  • Zotero Library is simply a link to the EthioStudies openly shared library in Zotero, which supports our bibliography.
  • List of cited publications is a searchable reverse index of the publications which have been cited somewhere in the dataset with a reference back to them.
  • Persons will show a list of marked-up persons in the data with a pie chart of the different attestations and links back to each occurrence.
  • Places will show a list of marked-up places in the data with a pie chart of the different attestations and links back to each occurrence.
  • Titles/Colophons/Supplications is a filtered search which will only look at this specific features in the manuscript descriptions.
  • Decorations is a filtered search which only looks at decorations in the manuscripts. Not all manuscripts have been marked-up with this information, you will only see what is available.
  • Bindings is a filtered search which only looks at bindings in the manuscripts. Not all manuscripts have been marked-up with this information, you will only see what is available.
  • Additions is a filtered search which only looks at additions in the manuscripts. Not all manuscripts have been marked-up with this information, you will only see what is available.
  • Keywords provides you access from the taxonomy in use by the project for the categorization of records.

Resources

Under this menu you can find direct links to additional visualizations which are also accessible from item pages. If you want to start from blank and use them, here is where to start from. We have also listed here some additional search options and specific projects part of the research environment.

  • Compare Manuscripts of a given work will bring you to a page where you can enter the identifier or the title of a Textual Unit and see all the manuscripts which contain it orderd chronologically.
  • Map of Manuscripts with a given content will offer you the possibility to look for a given Textual Unit and see charted in the DARIA-DE GeoBrowser the manuscripts which we know to contain this work.
  • Related Textual Units is a view requested by Prof. Wendy Belcher which will produce a huge table with information about the textual units related to one main one you may be intereste in. This relies on the fact that this information is available.
  • Literature Flow Sankey view is a view based on the presence of keywords which will group works or manuscripts containing these works and chart them to a sankey chart with intervals at each period of those defined within the project.
  • XPath search allows you to run any XPath search on the XML data and returns the items where the hit is matched.
  • SPARQL Endpoint allows you to run a SPARQL query to our RDF data and returns the results in several tabular formats (for the raw SPARQL requests, see the API Documentation).
  • Collate passages with Collatex ATTENTION! this should allow to run collatex on our server by pointing at data present in BM to compare. It works only if you notify us so we can check collatex is running.
  • Scholars in Ethiopian Studies will return with a bit of patience a summary of all scholars which are recorded in the database, with some additional informaiton and links for each.
  • Chonjacki Collection will link to a Mirador viewer instance gathering the images digitized during a collaboration project with the Vatican Library about this collection.
  • Tweed Collection will link to the splash page for this collection, curated by the team of the EMIP project.

Lists

Each of the lists accessed from the menu above is slightly different from the other, here we will give you some of the features of each of these. The main difference between these lists and the general search, is that these lists allow you to filter according to some features, and search based on those filters, while the General search will mainly look for text strings in specific places. A second difference between the lists and the search is that the results are presented differently. While some lists will give specific basic informations (those explained in this section), some others will contain all the content already and allow browsing and filtering (those under indexes, below). The General Search (see the searches section) will only offer a pointer to the resource matching the search parameters.

The lists views which are presented in this section share some features. On the left side you get filteres based on the indexes for that type of resources in the database, on the right side you will see, after searching, your results as in a table, paginated by 20 as per defaults. You can change the pagination option once you have done your search by entering a value in the "how many per page?" field of the pagination bar.

In the filter section on the left, some filters are not loaded, because they take too long to do so and are not always used. Each view will list them just before the buttons to run the search. Check one and the additional filter will load.

There are three buttons in the filter section: the search button, the one to return to the list view unfiltered, and one linking to the General search.

To get all records, run a saerch without setting any parameter.

Some parts of the inforamtion are loaded after the page is returned, they may take a bit.

The color of the lines in the results is meaningful, it tells you immediately if an entry is complete or not.

Once you have ruun your search, a bar with a summary of the results will appear at the top, with links to additional list views for the selected results and other actions for the entire list or for parts of it as selected. Clicking on the hints button, will produce some additional blinking buttons which will provide additional guidance on the content.

Both the available filters and the results depend on the list, and are expelained in the below sections.

Manuscripts

On a selection of manuscripts you will be able to get a direct link to the Charts view (see below), by selecting the specific manuscripts in the list returned, you will also be able to get directly from this view the PDFs, or to link to the Compare or Titles view for a selection of manuscripts in your result. Checking the box at the top of the relevant column will select all visible results.

Each result line will summarieze the formal name of the manuscript, the shelfmarks we know, the number of units, parts and hands, and the script. Some direct links to the source and some of the available views are also provided.

The values in the filters are also updated, so that the keywords available, for example are only those relevant for that selection, thus providing some summary information about the selection.

Frequently asked questions

You can always ask questions or make suggestions in our GitHub Issues where all the project discussions are publicly available. We you only the issues for that one repository, just to keep them all in one place. You can also see the contacts page.

Question Answer
Why is the text of this work not there? In the current phase of the project we focus on manuscripts from historical catalogues. We only enter information about literary works when this either part of collaborations or necessary for the catalogue entry. This means most of the time that Textual Units will only be identified with minimal information.
What transcriptions method do you use? This one. However, still plenty of text is transliterated according to the principles of the EAE. We change it as we go, so, there will be several system for quite a while. A reliable automatic conversion is not possible.
What is the difference between the witnessess and the manuscripts which contain a given Textual unit? The list of witnesses ideally contains only those manuscripts which have been used for one or the other edition. The list of manuscripts which we know contain this work may be different from these in several ways.
  • we may have a record, but this may still be not encoded with the reference to this textual unit,
  • we may not have a record for a manuscript which has been used as a witness for an edition,
  • we may have encoded a record of a manuscript which contains this work which has not been used for any edition yet.
Can I reuse this data? Yes, sure. You can download source, TEI, PDF, etc. and use them. Each file carries its attribution and licence, which you are kindly requested to pay attention to. Our data is versioned and furnished with permanent identifiers. If you want to cite a record, please use the permanent identifier of the version you are referring to.
