Some hints on how you can navigate this website

Introduction

This help page will walk you through what the website has to offer, additional explanations can be found on individual pages and clicking info buttons.

We will start from the navigation bar at the top of each view, then look at contents of each of the menus there in more details, discussing each list, item, resource and index view.

All this changes often, because we try to keep this Research Environment as flexible as it needs to be. For a discussion of how this website works, please see the links, presentations and bibliography below.

If you are looking for information about the project, please see our institutional project website.

This help page is all in one page, so you can search through it using your browser search function. The main parts are always accessible form the side bar.

Note that the data as well as the website are in continuous development. You are likely to encounter errors. If you care enough, please, do let us know and we will try to sort them as soon as possible or inform you on why this happens at least.

Navigation Bar

Login

You do not need to login. Only if you are an editor which uses the web based editor, then you might want to login. All content is served to you in its latest version, without any need to login.

Hi guest!

In this menu you find links to some documentation and general project information, especially the team, where you can see each contributor to the resources you are looking at and also navigate directly to a list of their contributions. Want to be among them? You can! The Guidelines have a page to get you started and also some training materials. There you will also get to know when there will be a next workshop.

Clavis

Under this menu you find links to three list views which relate to the Clavis Aethiopica , an ongoing repertory of all known Ethiopic Textual Units. You can use the provided CAe numbers to refer univocally to a specific text in your publications. Please note that this CAe reference number shares only the numeric part with the Textual Unit Record Identifier.

You can find more details on each of these lists and their visualization below. The three lists are:

  • Textual Units will return a list of what is most commonly thought of as a literary Work.
  • Narrative Units will return all identified Narrative Units. This are still very few.
  • Documents Corpora is a reproduction of a list view from the EMA project led by Anaïs Wion which lists encoded documents under archives.

Manuscripts

Under the manuscript menu you will find several ways to access lists of the available manuscripts.

  • Manuscripts (search) will lead you to a search form which will then list the manuscripts from the database with some information (more below)
  • Shelf marks (full list) will return a list of all repositories we know of and for each of them a list of shelf marks grouped by collection. If you know your shelf mark and you like scrolling, this is where to look. If you use the search without setting any filter, this will also look in shelf marks and get you to possible options.
  • Manuscript Images will load a Mirador viewer open on the list view, with all images of manuscripts we are currently serving or we have a usable link to a manifest for. You can either directly view the images in this mirador instance or click on the info box and go to the record for that manuscript.
  • Catalogues will provide you with a list of all publications which are listed in the Catalogue Bibliographies, and clicking on one of them will open a list view with all the manuscripts in that catalogue for which we have a record.

Places

Under the Places menu you can get to two different list views.

  • Places will lead you to a list view of all place-like records (including repositories).
  • Repositories will lead you to a list view of the repositories only and tell you how many manuscripts are kept there.

Indexes

The links listed under this menu will lead to additional views of the data according to specific needs. Each of these is explained in more detailes below in a specific section.

  • Zotero Library is simply a link to the EthioStudies openly shared library in Zotero, which supports our bibliography.
  • List of cited publications is a searchable reverse index of the publications which have been cited somewhere in the dataset with a reference back to them.
  • Persons will show a list of marked-up persons in the data with a pie chart of the different attestations and links back to each occurrence.
  • Places will show a list of marked-up places in the data with a pie chart of the different attestations and links back to each occurrence.
  • Titles/Colophons/Supplications is a filtered search which will only look at this specific features in the manuscript descriptions.
  • Calendar is a list of dates in the Ethiopian Calendar, which occur in the records.
  • Decorations is a filtered search which only looks at decorations in the manuscripts. Not all manuscripts have been marked-up with this information, you will only see what is available.
  • Bindings is a filtered search which only looks at bindings in the manuscripts. Not all manuscripts have been marked-up with this information, you will only see what is available.
  • Additions is a filtered search which only looks at additions in the manuscripts. Not all manuscripts have been marked-up with this information, you will only see what is available.
  • Keywords provides you access from the taxonomy in use by the project for the categorization of records.

Resources

Under this menu you can find direct links to additional visualizations which are also accessible from item pages. If you want to start from blank and use them, here is where to start from. We have also listed here some additional search options and specific projects part of the research environment.

  • Compare Manuscripts of a given work will bring you to a page where you can enter the identifier or the title of a Textual Unit and see all the manuscripts which contain it ordered chronologically.
  • Map of Manuscripts with a given content will offer you the possibility to look for a given Textual Unit and see charted in the DARIA-DE GeoBrowser the manuscripts which we know to contain this work.
  • Related Textual Units is a view requested by Prof. Wendy Belcher which will produce a huge table with information about the textual units related to one main one you may be interested in. This relies on the fact that this information is available.
  • Literature Flow Sankey view is a view based on the presence of keywords which will group works or manuscripts containing these works and chart them to a Sankey chart with intervals at each period of those defined within the project.
  • XPath search allows you to run any XPath search on the XML data and returns the items where the hit is matched.
  • SPARQL Endpoint allows you to run a SPARQL query to our RDF data and returns the results in several tabular formats (for the raw SPARQL requests, see the API Documentation).
  • Collate passages with Collatex ATTENTION! this should allow to run collatex on our server by pointing at data present in BM to compare. It works only if you notify us so we can check collatex is running.
  • Scholars in Ethiopian Studies will return with a bit of patience a summary of all scholars which are recorded in the database, with some additional information and links for each.
  • Chonjacki Collection will link to a Mirador viewer instance gathering the images digitized during a collaboration project with the Vatican Library about this collection.
  • Tweed Collection will link to the splash page for this collection, curated by the team of the EMIP project.

Search icon and info icon

Clicking on the search icon will lead you to the General Search. The search icon with the lightning, which is only partially faster, will link to the faceted search.

Lists

Each of the lists accessed from the menu above is slightly different from the other, here we will give you some of the features of each of these. The main difference between these lists and the general search, is that these lists allow you to filter according to some features, and search based on those filters, while the General search will mainly look for text strings in specific places. A second difference between the lists and the search is that the results are presented differently. While some lists will give specific basic informations (those explained in this section), some others will contain all the content already and allow browsing and filtering (those under indexes, below). The General Search (see the searches section) will only offer a pointer to the resource matching the search parameters.

The lists views which are presented in this section share some features. On the left side you get filters based on the indexes for that type of resources in the database, on the right side you will see, after searching, your results as in a table, paginated by 20 as per defaults. You can change the pagination option once you have done your search by entering a value in the "how many per page?" field of the pagination bar.

In the filter section on the left, some filters are not loaded, because they take too long to do so and are not always used. Each view will list them just before the buttons to run the search. Check one and the additional filter will load.

There are three buttons in the filter section: the search button, the one to return to the list view unfiltered, and one linking to the General search.

To get all records, run a search without setting any parameter.

Some parts of the information are loaded after the page is returned, they may take a bit.

The color of the lines in the results is meaningful, it tells you immediately if an entry is complete or not.

Once you have run your search, a bar with a summary of the results will appear at the top, with links to additional list views for the selected results and other actions for the entire list or for parts of it as selected. Clicking on the hints button, will produce some additional blinking buttons which will provide additional guidance on the content.

Both the available filters and the results depend on the list, and are explained in the below sections.

Manuscripts

On a selection of manuscripts you will be able to get a direct link to the Charts view (see below), by selecting the specific manuscripts in the list returned, you will also be able to get directly from this view the PDFs, or to link to the Compare or Titles view for a selection of manuscripts in your result. Checking the box at the top of the relevant column will select all visible results.

Each result line will summarize the formal name of the manuscript, the shelf marks we know, the number of units, parts and hands, and the script. Some direct links to the source and some of the available views are also provided.

The values in the filters are also updated, so that the keywords available, for example are only those relevant for that selection, thus providing some summary information about the selection.

Note that searching the name field will search the shelf marks of the manuscripts.

Examples:

  • manuscripts composed of 10 quires
  • manuscripts with at least 1 quinion
  • manuscripts with 1-100 leaves

Textual Units (Clavis)

While the filters are organized in a similar way to those in the Manuscript filtered search.

Clicking on the Voyant button in the top bar will transfer the entire text of the Textual Units in the selection to the web version of Voyant for full text analysis.

The results contain similarly some summary information about each Textual Unit (Work). The first column contain the Clavis Aethiopica identifier and a small table with corresponding identifiers for the same work from other Clavis.

In the titles column, which links to the record, all available titles are listed. while in authors all authors are listed, with a yellow highlighting in case of attributed authors.

As in the manuscript filtered search, you can select several entries or all the visible ones to print them as PDFs or send them to the manuscript map. This means that if from your selection you want to see on the map the locations of the manuscripts of two or more manuscripts, this is where to start, especially if you do not know the identifiers. If you for example search for "Miracle of Mary" you can select a series of miracles and see on a map for each miracle the manuscripts we know of in the ethiopic tradition which contain it, and see the distribution of each compared to the other. This is of course always limited to data availability!

The witnesses section will also list the available manuscripts with a direct link to them, and link directly to the comparison or map of manuscripts view.

Note that searching the name field will search in the titles of the Textual units only.

Places

The specific features of this list view are only that the results will also tell you if there is a Wikidata correspondence.

This data, as most of the data about Textual Units and Persons was extracted from the indexes of the Encyclopaedia Aethiopica, in Volume 5, and are still very much like that index. As we work through the data we correct and fix but there will always remain some inconsistencies.

Note that searching the name field will search in the place names of the place record only.

Persons

This filtered list as the previous has no specific feature, it simply returns some basic information about each person from a filtered search.

Note that searching the name field will search in the personal names of the person record only.

Repositories

The repositories filtered list will allow you to search across the repositories registered in the database, and return information exactly like the places filtered search. However there is a peculiarity here, and this is that the link to the repository will not lead you to an item view, but to an individual Repository view.

Individual Repository

The Individual Repository view blends in one page the information about the institution which we have, on the left side, which reproduce the same format as the place item view, and a list view identical to the manuscript filtered search view, but for space reasons, the filters are on the right and are already relevant to the manuscripts linked to that repository only.

You also get as a feature of the item view the bottom section with the citation, revision and attribution boxes.

Authority files

The list view for the database records which are linked to values in the taxonomy are provided without the ability to filter a search within these records. This may be offered in the future but the current data does not offer any useful feature here.

Instead, the organization of the terms in the taxonomy is reproduced to be browsed on the left. Clicking on each Category will pop-up its subcategories and clicking on the categories in there will produce a view of the entry followed by a list of all records which have that keyword associated to them. You can then either click on the title to go to the item view of that authority file, or on any of the resources associated to it.

This means also that if you want to saerch for records which have two keywords or more, this is not the place for you to start your search, you should use either one of the filtered searches or the General Search.

Charts

From a filtered list of manuscripts you can switch to a Chart view, where instead of the tabular information, some charts which use some of the data available in the selected records are offered. The same charts are also provided in the Comparison view.

Items

Item views represent one resource and its relations to others.

Some features are shared by most of the item views.

Under the navigation bar a list of optional views for the same item are offered. These vary according to availability. For example, if no transcription or text are available, this option will not be there.

There is always in this options bar a button called "Explore this page", which will activate a short step by step introduction to what the page has to offer. You find then a button to notify errors but also an edit button, which will allow you to edit the entry in question. You will be prompted to the GitHub file, which you can edit and make a PR with your changes. These will be reviewed, discussed with you and eventually approved.

You find then some options which you can use to facilitate your reading of the page, if needed, and show or hide the pointers from each linked entitiy and the bar on the right with all related items.

You find then a button to produce a PDF output from the entry and a button to the main entry, as well as one which will produce an enriched TEI file.

The following buttons are item specific and are discussed below.

The title line gives you basic name and references for each item, as well as authors of the record and their role. Under it you will have a small item navigation bar on the left, which can be scrolled and expanded for very long entries.

The content of the entry is than at the centre of the page and organized in different ways according to the item type.

On the right you have a list of possibilities to see related items, which also varies according to the item type. You will however always have a way to select one of the keyword associated to the current record and load a list of records which share that keyword. You will also always have a box for eventually existing public annotations made with Hypothes.is. This offers you a very easy way to interact and enrich the data visualized, by simply making an annotation which uses the tag suggested. We do not currently support any other annotation tool, but let us know what you use, and if it is possible we will fetch annotations from there as well.

always in the relations column on the right you will find all relations expressed in the current record and the links to the related entities, grouped by name of the relation.

One last shared feature of the item views is the bottom part, where you find human readable and actionable links to the RDF representations and a function to show all version of the resource and permalinks. If you click on any of this you will see the data at that point shown with the current interface, and the citation will be updated with the information available then.

The four boxes below provide Citation, Revision history (which can be entirely different from the version history!) and eventually present further information on the attribution of the contents, followed by a licence statement.

Most boxes are scrollable, and have a fixed height to manage better the space available.

Every item view, as well as the individual repository filtered list also have a function "Show Attestations", which will search for references to the current record and list them with some context information, for example, for a person, other persons which cooccur in the same context.

You can click on any linked entity to be brought to that item view, or click on the little hand to load items related to that entity.

We use Alpheios in an embedded form, so that you do not need to install the plugin. This means that any word in the languages supported can be double clicked to load morphological data and definitions from Alpheios. We provide Alpheios with a parser for Gǝʿǝz, so that you can double click on any word tagged as Gǝʿǝz and retrieve possible matches with links to the Online Lexicon Linguae Aethiopicae. Additionally, if you hover on any Gǝʿǝz word, we will offer a series of useful links to a search in the database for that word, a search in the dictionary, a search in the parser via its web interface and a search in the TraCES corpus of annotations.

Another function available from all items is the direct exportability of bibliographic metadata, which are embedded in the page.

Manuscripts main view

The main item view for manuscripts will provide you two main parts: the codicologial descriptions and the content description. You can hide either of them.

At any place in the description, which contains a placement in the manuscript for which there are photos, you can click to open a small viewer at the location in question.

Information is organized by Codicological Unit and contents are grouped, you will need to click on the button of an item to expand its content and see the information available for it.

If a collation is available, then the codicological part of the description will have buttons to display it, using VisColl.

Clicking on the title of a content will lead you to the Textual Unit record for that.

Additional views which may be available for a manuscript are the transcription view, the image view, the Syntaxe view and relations view, all detailed below.

Textual and Narrative Units main view

Landing on the main view for a Textual unit you will see the main title, but also the Clavis Aethiopica number, the responsible editors and authors of the record in question and the status of the resource.

The main content is in this view more straight forward, as it provides the information availble about the text. If a text or even just a part of it is availble, you will see a red button in the options bar just under the navigation bar.

In the item navigation bar, you will also have direct links to the places and persons list views already filtered for this work. This is different from what you will get clicking the places optional view for a work item, which will print on a map the places attested and marked up in the given resource.

The witnessess section will contain links to manuscripts used for the edition from which the record was created.

At the bottom of the main content area you will have, beside the attestations button also a Voyant button, which will transfer the entire data available into Voyant web version, for your further analysis needs.

The relations bar on the right is topped by the list of manuscripts which we know to contain the given work. A link is provided to see these in the Manuscript filtered search view. Any Textual Unit can appear either alone or as part of others and we will list here also those manuscripts which contain a text which contains the current Textual Unit. For each matchin manuscript we will tell you the name, linking to the record, if there are images, and also extract some data about placement of the Textual Unit in the manuscript, with the identifier of the item, the placement, the position in the count of items, and if there is any text available, a word count for it.

Places main view

The item view for places features at the top links to classification and filtered searches for the place classification, as well as a link to searches for manuscripts to be found in repositories connected to that place.

Two maps are provided when available, one based on the coordinates we have, the other from Pelagios Commons API.

The item navigation bar will also directly link to the places index filtered for this entity.

The main data section provides information as it available, the relation bar on the right additionally provides a list of churches and monasteries of which we know about and which are related explicitly with this place. Also a list of resources linked to this place in Pelagios is added.

Persons main view

What distinguishes this item view from the others is the presence of a box with the core informations available: names, relevant dates, etc.

The item navigation bar will also directly link to the persons index filtered for this entity.

Authority files main view

These only contain the general information and the relation as above and have no additional specific feature.

Text/Transcription view

For manuscripts and textual units, you may have respectively a transcription or a text available, more often just parts of either. We welcome contributions in this sense, if you have content which can be shared and has clear sources.

The text and transcription view is divided in three main areas, one with the witnesses (for text), the area of the text and eventually images related to it as well as apparatus, and the bibliography.

The first level of the organization of the text is used to paginate, but you can always show the entire text clicking on "See full text". In the area with the text the transcription will produce the features encoded (e.g. red for rubrication, switching alternative reading, etc.) while the edition text, if annotated, will have apparatus notes linked to the side.

Each structural unit of the text may also have links to quotations of that passage, to an alignment tool for different version and to alternative versions if available. An info button will tell you more about each.

If a translation of the passage viewed is available, you will see that as well. If images of the witness manuscripts are available and explicitly linked, the images of all available linked witnesses will be shown below the text transcription and follow on with the navigation. There is currently only one partial example of this however, and we hope to have more in the future.

Compare

Textual Units and Narrative Unit, will have a button in the option bar, linking to the Comparison view. This will print for a given Textual or Narrative Unit, all the manuscripts which contain it with a summary of their contents ordered by date. You will see the unit in question highlighted to enable you to see where it occurs in the different manuscripts. Under this you will also have charted information from the manuscripts in this selection.

Graph

In most cases this view does not do anything yet. It will pull information useful from the RDF representation of the data, when it is available. Because the RDF only contains some of the information encoded in the TEI, this is often nothing additional to show here.

Syntaxe

In this view we provide tool for the structural analysis of manuscripts following La Syntaxe du Codex, namely a table to support the identification of convergent discontinuities and a Sankey chart summarizing the stages of transformation and circulation of a given manuscript.

PDF

This view will produce from the contents a formatted PDF with most of the available information.

TEI/XML

This view will produce an enriched and explicit TEI file for interchange and reuse.

Relations

The relations view of any item will produce a force graph of the entities and their relations to the current one, as well a table of such statements including statements which point to the current resource.

Images Viewer

For manuscripts of which we store and serve images, the images view will offer a Mirador viewer reading a IIIF Manifest extracted from the TEI. This allows the creation of ranges which make the navigation of the set of images easier, by linking a feature, for example a decoration to the exact image. The information box in the viewer will also provide additional information about the images.

geoJson

For places records where sufficient information is at hand, it will be possible to see or retrieve also a geoJson formatted version of the data.

Indexes

Some specific features of the records can be searched as such in the database, with dedicated indexes dedicated to those who are predominantly interested in these features.

List of cited publications

This is a reverse index of all publications actually cited in the database. For each bibliographical reference you will have the Zotero tag, and a link to the EthioStudies library and a list of all entries pointing to this. Eventually, if you filter this by catalogue bibliography, then you will have the same as the Catalogue list view.

You can limit this list to a specific type of record (e.g. only Textual Units). Clicking on the link which you find after each bibliographic record in the item views, you will land at this view filtered for that record only.

List of places/persons annotations

This indexes, requested initially by Éloi Ficquet, provides a paginated list of places annotated in the records, anywhere, so, either inside the text or in its description. For each place you will see each annotation listed and grouped by record in which it occurs. Beside the list a pie chart with the break down of the actual forms annotated is also provided. You can limit to a specific identifier or to a collection.

Titles Supplications Colophons

This filtered view will give access to what is marked up as title, colophon or supplication in the records.

You will be returned directly a list of all relevant features on loading the page, with specific filters. Each filter refers to features of the selected items. The full text of the titles colophons or supplications is provided in the result list, with a reference to the element where it occurs and a link to the resource.

Calendar

This view also offers access to parts of texts which have been associated to a relative calendar date, for example for liturgical recurring celebrations.

You will be able to limit your search by day or month and see in which records there are elements which have been associated with such calendar date.

Decorations

The decorations filtered search, originally designed with Jacopo Gnisci, will look at decorations and their features only. The filters are relative only to the selected features, reading the legends will help you to figure out what you can filter. For example you can search for all encoded decorations of a specific art theme, or search the encoded legends. If the decorations are present, but not encoded, you will not get them in the results.

If an image is available, you will also find a thumbnail linking to the image viewer for that manuscript.

Bindings

The same type of filtered search is offered here for bindings and their features. Eliana Dal Sasso designed the encoding and the search for binding features present here.

Additions

Here you will be able to access and browse the text of all the texts which are recorded as additions to manuscripts.

Resources

Compare

The comparison view will take as input any Textual Unit identifier and search for the manuscripts which contain it and have been encoded with a proper identifier. See above the comparison item view.

Map of Manuscripts

This map view, using the Dariah-De Geobrowser will extract from manuscript records which have been explicitly linked to a Textual Unit, information on their current repository and original location as well as date. This is as always subject to data availability, but if you are not satisfied with the data, you can also add your contribution.

When the Collatex is running on the server, you can use the application to compare texts which are stored in the database. The results will highlight segments which are equal or not in a tabular view.

Chojnacki collection

Jacopo Gnisci worked on the Chojnacki collection at the Vatican Library to digitize photos donated by the scholar to that library. Clicking at this link you will see these images as they are provided via IIIF by the Vatican Library. This work was supported by Beta maṣāḥǝft.

Tweed collection

Some examples

  • If you search Taammera, you will not find Taʾammǝra or Taʿammera but only Taammera. Try Ta`ammera instead or use the keyboard provided to enter aleph and ayn.
  • If you are searching for Yāʿǝqob, you will not have a lot of luck searching Yaqob, unless some kind cataloguer has actually added it into the data as simplified spelling form. Try instead entering Yaqob~0.5 which is a fuzzy search, this will return also Yāʿǝqob. Also Ya`eqob is fine for example.

Hold and choose

If you hold a key optional values will appear in a list. You can click on the desiderd value or use arrows and enter to select it. The options are the same as those activated by combinations.

With this method you do not have to remember or lookup combos, but it does take many more clicks...

Basic search (home page)

From the homepage you can run a simple search which will look in all data but only in some elements. This is a most simple search that we can offer: it searches your text only in a few most common elements, and just as you type it.

Little options, little trouble, very sintetic results, i.e. you really need to know already what you are looking for.

Examples:

  • Search the word 'simple'.

Faceted Search

This new faceted search allows you to search for a term wherever it is, and then drill down the results, as in a commercial website. Once your hits have been retrieved you can select the features which help you restrict to a set of results. Beware, however, because not everything can be indexed as text. if you search for "Mary", you will not match all instances, but just those where Mary is written up in the text. In most cases, this is not the case, as we prefer where possible to use a pointer and store the name in an authority file only. It is true that you will match that person file, and from there you can navigate, and check occurrences and attestations of a given person, for example. In this search, we are not trying to give you sliders and similar things, or format the contents, or filter out dirty data, you have to declaratively select among the available values, for types of decorations as for dates.

Examples:

  • Search the word 'Mary'.
  • Search the word 'Mary' and filter down for manuscripts only with at least one decoration with art theme "Virgin and Child" (AT1031VirginAndChild).

General search

The general search offers a series of filters which affect all records, which are all in TEI. You can narrow down to one or some of the types and you can apply multiple filters. The info boxes will also support the use of this search functionality.

As in the filtered searches, there will be some filters, which need to be loaded before they can be used. You will have to select the type of file and then select one of the filters.

The results will be grouped by type of resource and they will only link to the entry, but some context will be provided for the first three matches in each record.

If it is not only a text based search which you want to do, you may find some of the options in the advanced search more suited. You can avoid searching text, which is one thing you cannot do in other searches. But if you search for text, here you can additionally specify which elements to search and restrict to a specific collection beforehand, for example. Because the filters are offered for the entire dataset, you may have to request them first, selecting from available menus.

Examples:

  • Search works with Textual Units tagged with keyword 'Liturgy'.
  • Search in the TEI element 'measure' for the number 5.
  • Search 'Makʷannǝn' in persName elements of person records, where a relation with name betmas:husbandOf is present..

This page also includes special requests, like the one for role. Because roles are assigned to references to a person and not in a person record, this is not going to filter your hits to those which have a persName with type role. It would not make that much sense. If you select a role the engine will return you all references to any person with that role and a link back to the records where they are mentioned with that role. You don't need to hit search, just hold on and they will appear on the right.

XPath search

All searches are XPath searches in an XML database like the one behind this website, but what you can ask in a XPath query is much more than any of the provided search interfaces can offer. So, if you know how to write your XPath, and know the source TEI (available for each file, by appending .xml to the identifier of the record) you will then be able to see your results listed.

Here you can avail yourself also of the structure of the file, the relative position of elements, and the nested elements, querying across the axes.

Examples:

  • person record which have at least some attribute for birth and death (can be when, notBefore, notAfter) elements and occupation type ruler'.
  • Manuscripts with an addition element typed Ownership Note followed by another one with type Supplication.

SPARQL Endpoint

Parallel to the XML, also an RDF triple store is maintained by the project. Here you get an interface to the SPARQL endpoint. You can add your SPARQL query and see the results available. The endpoint can be queried from other visualization tools like Palladio.

Examples:

  • Search for female donors.
  • Manuscripts with a patron of the imperial family
  • mountains mentioned in Liturgy manuscripts

Citation

To cite pages in the website, look first at the bottom of the page. versioned resources have long lasting links . If not, you have a number of other options depending on what you want to do.

Simply point to the general page (e.g. https://betamasaheft.eu/gender) or to one of the views (/gender/table, /gender/graph, /gender/page) or to the data at /gender/data. All these are stable links, they will not change. Their content will, these are not resources subject to versioning, like manuscripts, e.g., which can be cited using PIDs, but only visualisations of aggregates and queries.

To cite a stable version of the specific dataset (/gender/data) which you used via its HTML view, so not the latest version and not that specific view, but indeed what you used in whatever way, you could save a copy of the data, in a format of your choice to Zenodo, and point to that dataset and its DOI, which allows very easy versioning of the imported resources in case you need to update that deposit. The metadata would have to contain reference to BM as required by the CCBYSANC of the data, allowing non commercial derivates with attribution.

If you want to cite the page as it looks when you are looking at it and with the data as it looks, save it to the Way Back Machine with any of the available plugins and cite the web resource pointing to the version stored there. e.g. https://web.archive.org/web/20200812190615/https://betamasaheft.eu/gender/table/female, this adds to the declaration of when you looked at something the possibility to check it at that point in time.

Frequently asked questions

You can always ask questions or make suggestions in our GitHub Issues where all the project discussions are publicly available. We you only the issues for that one repository, just to keep them all in one place. You can also see the contacts page.

Question Answer
Why is the text of this work not there? In the current phase of the project we focus on manuscripts from historical catalogues. We only enter information about literary works when this either part of collaborations or necessary for the catalogue entry. This means most of the time that Textual Units will only be identified with minimal information.
What transcriptions method do you use? This one. However, still plenty of text is transliterated according to the principles of the EAE. We change it as we go, so, there will be several system for quite a while. A reliable automatic conversion is not possible.
What is the difference between the witnesses and the manuscripts which contain a given Textual unit? The list of witnesses ideally contains only those manuscripts which have been used for one or the other edition. The list of manuscripts which we know contain this work may be different from these in several ways.
  • we may have a record, but this may still be not encoded with the reference to this textual unit,
  • we may not have a record for a manuscript which has been used as a witness for an edition,
  • we may have encoded a record of a manuscript which contains this work which has not been used for any edition yet.
Can I reuse this data? Yes, sure. You can download source, TEI, PDF, etc. and use them. Each file carries its attribution and licence, which you are kindly requested to pay attention to. Our data is versioned and furnished with permanent identifiers. If you want to cite a record, please use the permanent identifier of the version you are referring to.
Why there is no English title for that? Titles are all computed, on the basis of rules of priority in extracting and building the best possible title for each resource. If a title in English exists but is not marked as a main title, this will not be considered as important and distinguishing as an available transcription of an ethiopic title. This may also just be due to the fact that nobody has yet added that label, and you can do it yourself if you want.
What is the fastest way to get the result? Each search function is stronger on some aspects, so it really depend what you are looking for. For text based queries, the faceted search is better. The database does not contain much text however, compared to metadata.
